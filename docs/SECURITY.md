# Security Standard

- Store secrets only in environment variables or a secrets manager.
- Never commit `.env`.
- Never paste private keys into source code, logs, issues, or chat.
- Use separate development and production credentials.
- Use least privilege and documented key rotation.
- Use a private GitHub repository with multi-factor authentication.
- Logs must not contain account numbers, tokens, keys, or authentication headers.

Phase 1 requires no Robinhood credentials.
